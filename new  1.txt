Produkt-Idee

Kurze Einleitung (Stichwort artig)
* 	Jeder Spieler schlüpft in die Rolle eines Avatars / Charakters, ein rollenspiel-typisches Element, jeder Charakter kann durch verschiedene Skills individualisiert werden. 
* 	In einem Schlachtfeld kann der Spieler in verschiedenen Spielmodi antreten (Deathmatch, Team-Deathmatch, Capture the Flagg, Base-Captureing) 5vs5, 10vs10 oder 15vs15
*	Modi für un-/gewertete Schlachten (Unranked & Ranked Battles)
*	Durch eine Wertung spielt der Spieler immer gegen etwa gleichstarke Mitspieler
*	Spieler erklimmt keine Level, sondern erhält eine Spielerwährung für gewonnene Spiele
*	Mit der Spielwährung werden Items oder Skills gekauft.
* 	Verschiedene Klassen um den Spielstil zu individualisieren, Klassen sind mit "Grundfähigkeiten" ausgestattet.
*	Belohnungssystem durch Awards (10 gewonnende Spiele hintereinander) andere Spieler sehen diese Awards
*	Erholungssystem, wenn ein Spieler längere Zeit nicht spielt, dass die Belohnung um den Faktor X größer sind
*	Ingame-Tuniere, wobei man besondere Awards oder Auszeichnungen bekommt

Architecture und Technologie - Overview
*	Server-Client-Architektur mit Kommunikation über JSON
*	Server geschrieben in Java (universal)
*	Client geschrieben in HTML5, JavaScript, CSS3
*	Backend mit Java und OpenSource-Datenbank (Firebird, MySQL, Derby)
*	Java-Server baut auf Sockets auf, so ist eine TCP/IP Verbindung garantiert mit möglichst wenig "Overhead"
*	Client fragt alle n-Millisekunden Daten vom Server ab und aktualisiert so alle Spieler
*	Client auslegen für Android, iOS, Web-Application, dass auf jeder Plattform gespielt werden kann und somit eine breitere Masse von Spielern getroffen wird
